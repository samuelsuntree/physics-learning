LLM 进化算法 ↔ 统计物理的类比

  先厘清你的系统的核心特征

  一个典型的 LLM-as-mutator 进化算法大概是这样的：

  1. 种群：一组 policy（比如代码片段、策略描述、prompt 等）
  2. 适应度：某种评估函数打分
  3. 选择：保留高适应度个体
  4. 变异：用 LLM 对个体做改写/重组，生成新个体
  5. 迭代：重复以上过程

  关键问题：这个系统像什么？

  不太像 Ising 模型

  Ising 模型的核心特征是：
  - 离散格点上的自旋，每个只取 $\pm 1$
  - 近邻相互作用：能量由相邻自旋的对齐/反对齐决定
  - 状态空间简单且结构化

  而 LLM 进化算法中，个体是高维、复杂、非结构化的对象（一段代码、一个策略），个体之间没有"近邻"的
  空间结构，变异也不是简单的翻转一个 bit。Ising 模型太简单了，无法捕捉 LLM 变异的丰富性。

  Ising 模型有一个地方可以类比——模拟退火（simulated annealing）。模拟退火就是在 Ising 类模型上跑
  Metropolis 算法，用温度控制接受劣解的概率。如果你的进化算法中有类似"温度"的参数控制 LLM
  变异的激进程度，那这个局部类比是成立的。

  最自然的类比：正则系综

  我认为正则系综是最贴切的类比。理由如下：

  类比对应表：

  ┌────────────────────────────────────────────┬───────────────────────────────────────────┐
  │            统计物理（正则系综）            │               LLM 进化算法                │
  ├────────────────────────────────────────────┼───────────────────────────────────────────┤
  │ 微观态 $i$                                 │ 一个具体的 policy                         │
  ├────────────────────────────────────────────┼───────────────────────────────────────────┤
  │ 能量 $E_i$                                 │ 负适应度 $-f_i$（适应度越高，"能量"越低） │
  ├────────────────────────────────────────────┼───────────────────────────────────────────┤
  │ 温度 $T$                                   │ 控制变异/选择激进程度的参数               │
  ├────────────────────────────────────────────┼───────────────────────────────────────────┤
  │ Boltzmann 分布 $P_i \propto e^{-E_i/k_BT}$ │ 选择概率 $P_i \propto e^{f_i/T}$          │
  ├────────────────────────────────────────────┼───────────────────────────────────────────┤
  │ 配分函数 $Z$                               │ 归一化常数 $\sum_i e^{f_i/T}$             │
  ├────────────────────────────────────────────┼───────────────────────────────────────────┤
  │ 热平衡态                                   │ 算法收敛后的稳态种群分布                  │
  ├────────────────────────────────────────────┼───────────────────────────────────────────┤
  │ 自由能 $F = E - TS$                        │ 算法隐含优化的目标：适应度与多样性的平衡  │
  └────────────────────────────────────────────┴───────────────────────────────────────────┘

  最后一行是最深刻的。自由能 $F = E - TS$
  意味着系统不是单纯地最小化能量（最大化适应度），而是在能量和熵之间做权衡。低温时能量项主导，系统
  趋向基态（贪心选最优）；高温时熵项主导，系统趋向无序（随机探索）。

  进化算法的核心困境——exploitation vs exploration（利用已知好解 vs
  探索未知区域）——恰好就是自由能中 $E$ 项和 $TS$ 项的竞争。

  温度的具体对应

  在 LLM 进化算法中，"温度"可能体现为多个参数的组合：

  - LLM 的 temperature 参数：直接控制变异的随机程度
  - 选择压力：tournament size、top-k 截断比例等
  - 变异的 prompt 指令：要求 LLM "做微调" vs "大胆改写"

  这些都在扮演统计物理中温度的角色：控制系统在适应度景观上的探索范围。

  为什么不是巨正则？

  巨正则系综要求粒子数涨落——对应到进化算法就是种群大小在变化。如果你的算法中种群大小是动态的（比如
  好的个体会"繁殖"更多后代，差的被淘汰，种群总数不固定），那就有巨正则的味道。但大多数进化算法的种
  群大小是固定的，所以正则系综更合适。

  ---
  如果想做得更深：自由能的视角

  正则系综类比给你一个非常有力的理论工具。定义：

  $$\mathcal{F}(T) = -T \ln \sum_{i \in \text{population}} e^{f_i / T}$$

  这个"自由能"有如下性质：

  - $T \to 0$：$\mathcal{F} \to \max_i f_i$（纯贪心，选最优个体）
  - $T \to \infty$：$\mathcal{F} \to \langle f \rangle - T \ln
  |\text{population}|$（纯随机，均匀采样）
  - 中间温度：自动在两者之间权衡

  你可以用这个框架来理论分析最优温度调度——类似模拟退火的冷却策略。开始时高温（广泛探索），逐渐降温
  （聚焦到好解附近）。

  ---
  更激进的类比：LLM 作为热库

  还有一种思考方式更深一层：

  在正则系综中，热库的作用是提供随机的能量涨落，让系统能够跳出局部最小值。LLM
  在这个进化算法中扮演的角色恰恰就是热库——它提供"智能的随机涨落"。但它比物理中的热库更强大：物理热
  库只提供无方向的随机涨落，而 LLM 提供的是有语义结构的、有方向的变异。

  这意味着 LLM 进化算法实际上对应一个非平衡统计力学系统——热库不是无结构的白噪声，而是有记忆、有偏
  好的"智能热库"。这比标准统计力学复杂得多，但也意味着它可以比标准的 Metropolis-Hastings
  采样高效得多——LLM 的语义理解让它能够直接提议高质量的变异，而不是盲目随机游走。
