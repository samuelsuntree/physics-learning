## B. LLM调用机制

- OpenAI兼容API (src/openevolve/llm/openai.py:108-226)
- LLMEnsemble权重随机采样 (ensemble.py:70-73)
- 当前配置: DeepSeek-chat单模型
- Prompt构造: system(角色) + code + metrics + history + inspirations (sampler.py:51-172)
- V4岛屿系统消息覆盖 (island_novelty_worker.py:147-153)
- generate_multiple()存在但未被调用——没有best-of-n

## C. 进化循环

- 主循环: island_novelty_parallel.py:107-437
- 种群初始化: 仅单个初始程序 (controller.py:278-326)
- 选择: 三模态 或 temperature-softmax (island_novelty_database.py:356-440)
- 变异: LLM diff + 5%架构变异全重写 (island_novelty_worker.py:104-121)
- 交叉: inspirations隐式 + 跨岛灵感注入 (island_novelty_database.py:393-423)
- 终止: max_iterations=300 / target_score / early_stopping_patience

## D. 适应度评估

- 级联: Stage1(3ep快速) → Stage2(5ep+video+Gemini)
- 核心: combined_score = mean_reward (evaluator.py:324)
- 半确定性: 固定种子seed=42+i, 可选噪声注入(当前关闭)
- 多岛不同目标: Island0=fitness, Island1=novelty_elo

## E. 种群管理

- population_size=500, archive=50
- MAP-Elites: complexity维度, cell_capacity=15, 阈值[500,650,750,850,1000,1200]
- 2岛 + 每50代迁移10%
- Novelty ELO: 三轮漏斗 + Swiss锦标赛
- 无显式mode collapse检测，间接防范: 多岛 + exploration + 架构变异 + similarity_threshold=0.99

## F. 与统计物理的映射

### F.1 微观状态 → Program对象
状态空间是离散、极高维、非结构化的代码字符串空间。需要定义合适的序参量来降维。

### F.2 能量函数 → combined_score
E = -combined_score。多能量面雏形：不同岛屿用不同"能量函数"(_get_island_score(), 行184-203)。

### F.3 多层温度结构
1. LLM temperature (0.8) = 采样温度，影响代码生成分布
2. Sampling temperature (1000) = 选择温度，rank-based softmax (行507-545)
3. Reward temperature (exploration_ratio*10+0.1=2.1) = 混合采样温度
4. Novelty judge temperature (0.1) = 近确定性测量
5. Exploration/Exploitation比例 = 离散混合温度

### F.4 三层Boltzmann结构
- 层1: calculate_softmax_heat的双系综线性混合(α=0.7)
- 层2: ELO系统的Fermi-Dirac分布 (β=ln(10)/400)
- 层3: rank-based softmax的无量纲化处理

### F.5 配分函数
Z = sum_exp (island_novelty_sampling.py:16) 和 total (island_novelty_database.py:539)
当前只用于归一化，自由能 F=-kTlnZ 未被计算利用。

### F.6 相变信号
- 架构变异(5%) = 一阶相变类比
- Novelty ELO Rerank = 可能的排名剧变
- Migration = 突然改变岛屿基因组成
- 缺失: 没有监控多样性指标随迭代的变化

### F.7 系综映射
- 微正则 → 同fitness程序集
- 正则 → temperature采样分布
- 巨正则 → 动态种群(有上限500)
- 多重正则 → 多岛屿系统

## G. 改进建议

### G.1 统一Boltzmann选择
位置: island_novelty_database.py:442-505
定义统一自由能 F = -fitness + λ·novelty，用Boltzmann分布 P(i) = exp(-βF_i)/Z 采样。

### G.2 退火调度
位置: island_novelty_parallel.py:107+ 主循环中
指数退火 T(t) = T_max · α^t + 配置类新增annealing参数 (island_novelty_config.py)

### G.3 统一能量函数
位置: island_novelty_database.py 新增 compute_energy()
E = -fitness + λ_complexity·complexity_penalty + λ_similarity·centroid_distance

### G.4 相变检测
位置: island_novelty_parallel.py:348+ checkpoint附近
计算比热 C_v = Var(E)/T², 监控磁化率(dominant architecture fraction)

### G.5 Ising模型分析
位置: evaluator.py 新增 compute_ising_energy()
将6关节二值化为自旋，计算耦合矩阵J_ij和磁化强度m

### G.6 完整系综框架
建议新建 ensemble_physics.py 模块，实现正则系综采样器+退火调度器+自由能计算器+相变检测器+副本交换方法
