## 第一部分：跨学科深度对话

*以下为物理学家（P）与大模型科学家（M）围绕七个核心话题的深度讨论。*

### 话题一："LLM本身就是一个能量基模型"这个发现意味着什么？

**P：** Blondel等人2024年的工作给出了严格证明：自回归语言模型（ARM）和能量基模型（EBM）之间存在显式双射。本质上，当你把一个自回归模型的条件概率链式展开时，你可以把整个序列的联合概率写成 $P(x) = \exp(-E(x))/Z$ 的形式——这恰好就是Boltzmann分布。这里 $E(x)$ 是一个隐式定义的能量函数，而 $Z$ 是配分函数。这不是类比，这是数学等价。

**M：** 同意数学等价性，但我想强调工程含义。当我们让DeepSeek V3生成代码变异时，temperature=0.8这个参数，在物理图像下就是 $\beta = 1/T = 1.25$。这意味着我们的采样是在一个真实的Boltzmann分布上进行的——只不过这个"能量景观"是由模型的训练隐式定义的。Boltzmann-GPT（2026）的"嘴巴不是大脑"架构走得更远——用深度Boltzmann机（DBM）作为世界模型来定义能量景观，然后用自回归头来采样。

**P：** 对，而且2026年"Beyond Normalization"那篇新论文把配分函数重新解释为RLVR中的难度调度器，这一点非常深刻。在我们的系统里，配分函数 $Z = \sum_x \exp(-\beta E(x))$ 实际上编码了整个搜索空间的"可达性"信息。当 $Z$ 很大时，说明有很多低能量（高质量）的解容易被采样到；当 $Z$ 很小时，说明好的解稀疏且难以找到。这个量在当前代码里被计算了但被丢弃了（`calculate_softmax_heat` 函数中的 `sum_exp` 就是 $Z$），这是一个重大损失。

**M：** 更进一步说，NVIDIA的Energy-Based Diffusion Language Models展示了另一条路线：不通过自回归采样，而是直接在能量函数上做扩散。这对我们的进化系统有启发——如果我们能显式地定义程序空间上的能量函数，就可以用扩散过程来直接"生成"好的变异，而不是依赖LLM的隐式分布。

**P：** 但这里有个根本性的张力：LLM的隐式能量景观和我们定义的适应度景观是两个不同的东西。LLM"认为"什么是好代码，和这个代码在HalfCheetah上跑出什么分数，这两个能量面没有先验理由是对齐的。Soft Bellman方程给出了一个联系——当ARM通过RLHF/RLVR训练后，它的隐式能量面会偏向奖励函数——但对于代码进化这个任务，这个对齐程度是不够的。

**M：** 这正是我想说的关键洞察：当前系统中存在**双重能量景观**。一个是LLM内部的（决定它倾向于生成什么样的代码），另一个是评估器定义的（MuJoCo reward）。进化过程其实是在试图让采样从LLM的能量面转移到评估器的能量面上。Temperature、选择压力、种群动力学——所有这些都是在调节这两个能量面之间的关系。

### 话题二：当前系统中已经隐含了哪些统计物理结构？这是巧合还是必然？

**P：** 我仔细审查了代码，发现至少存在四层Boltzmann结构，而且工程上写成了几乎完全对应物理公式的形式。这不是巧合。

第一层：`calculate_softmax_heat()` 函数（`island_novelty_sampling.py`第8-21行）。这就是 $P_i = \exp(s_i/T) / Z$，纯粹的Boltzmann分布。参数 `temperature=1000` 对应物理温度 $T$。

第二层：ELO系统的 `expected_score()` 函数（`novelty_elo.py`第30-32行）。公式 $E_A = 1/(1+10^{(\text{elo}_B - \text{elo}_A)/400})$ 在数学上等价于Fermi-Dirac分布 $f(\epsilon) = 1/(1+\exp(\beta(\epsilon-\mu)))$，其中 $\beta = \ln(10)/400 \approx 0.00576$。

第三层：`_sample_from_island_weighted()` 中的双系综线性混合（`island_novelty_database.py`第442-505行）。这个 $\alpha \cdot P_\text{reward} + (1-\alpha) \cdot P_\text{novelty}$ 的混合，严格来说是两个正则系综的线性组合——这在物理上对应着广义Gibbs系综（GGE）的一个特殊情况。

第四层：多岛系统本身。三个岛（fitness/novelty_elo/elegance）以不同的"竞争规则"（即不同的Hamilton量）运行，并通过迁移交换个体。这正是并行退火（parallel tempering）或副本交换Monte Carlo的结构——只不过目前的迁移（`migrate_programs()`第614-693行）缺少Metropolis接受准则，所有迁移都无条件接受。

**M：** 我认为这是**必然的**。让我解释为什么。任何在离散对象的大集合上做"选择好的、丢弃差的"的操作，如果你加了随机性（temperature），你就自然得到了Boltzmann分布。FunSearch的程序池、MAP-Elites的archive、我们的岛屿系统——它们都是正则系综的实例。真正的问题不是"它们是不是Boltzmann分布"——它们一定是——而是"我们有没有正确地利用这个结构"。

**P：** 完全同意。而当前系统的问题恰恰在于，虽然各个组件单独看都是正确的Boltzmann分布，但它们之间**没有热力学自洽性**。具体来说：
1. LLM的temperature（0.8）和采样的temperature（1000）没有任何关系——前者控制文本生成的随机性，后者控制父代选择的随机性，这两个"温度"在物理上是不同系统的温度。
2. 配分函数 $Z$ 在 `calculate_softmax_heat` 中被计算（第17行的 `sum_exp`）然后被丢弃，失去了携带的热力学信息。
3. 岛间迁移没有Metropolis准则，破坏了详细平衡，意味着系统不会收敛到任何稳态分布。

**M：** 你说的"热力学自洽性"我理解，但我想辩护一下工程决策。采样temperature设为1000是因为我们对原始分数做了rank-based softmax（`_sample_from_elite_cells_temperature`第527-545行），ranks的范围是1到N，所以temperature=1000在rank空间中实际上对应一个非常"热"的系综——几乎均匀采样。这是为了在进化早期保持探索。而如果这个temperature像物理退火一样随时间降低，我们就能实现从探索到利用的平滑过渡。

**P：** 对，这就是模拟退火。而且理论保证的对数冷却调度 $T(t) = T_0 / \ln(1+t)$ 对于收敛到全局最优是充分必要的。但在实践中，指数冷却 $T(t) = T_0 \cdot \alpha^t$（$\alpha \approx 0.95$）更常用，虽然没有理论保证，但收敛速度快得多。

### 话题三：系综理论能给进化搜索带来什么本质性的改进？

**P：** 让我给一个定量论证。在统计力学中，自由能 $F = U - TS$ 给出了"能量"和"熵"的最优权衡。把这映射到进化搜索上：

$$F_\text{free} = \langle f \rangle - T \cdot S_\text{pop}$$

其中 $\langle f \rangle$ 是种群平均适应度，$S_\text{pop} = -\sum_i p_i \ln p_i$ 是种群分布的熵（多样性度量）。

**这个公式告诉我们：最优的进化策略是最小化自由能，而不是最大化适应度。** 当温度 $T$ 高时，熵项主导——系统应该探索；当 $T$ 低时，能量项主导——系统应该利用。这给了我们一个原则性的框架来做exploration-exploitation权衡，而不是像现在这样用硬编码的比例（exploration_ratio=0.2, exploitation_ratio=0.7）。

**M：** 这和SAC（Soft Actor-Critic）中的最大熵RL是完全一致的。SAC的目标就是最大化 $\sum_t \mathbb{E}[r_t + \alpha H(\pi_t)]$——奖励加上策略熵。$\alpha$ 就是你的温度 $T$。而且SAC已经证明了这个框架在连续控制任务上的有效性。

但我想指出一个**范式性的改进机会**：GFlowNet。GFlowNet的核心思想是训练一个生成模型，使得它的采样概率正比于奖励：$P(x) \propto R(x)$。这恰好就是Boltzmann分布 $P(x) = \exp(-\beta E(x))/Z$（令 $R(x) = \exp(-\beta E(x))$）。ICLR 2024 Oral上Hu等人的工作"Amortizing intractable inference in LLMs"已经展示了如何把LLM微调为GFlowNet——让它直接按照目标分布采样，而不是通过进化来间接趋近。

**P：** GFlowNet确实是理论上最优雅的方案，但我有两个担忧。第一，它需要微调LLM，这在我们的设置中成本很高（我们用的是DeepSeek V3 API）。第二，Prugel-Bennett 1994年的遗传算法统计力学分析告诉我们，进化算法的种群动力学本身就可以用宏观序参量（如种群的平均适应度和方差）来精确描述——这些宏观量的动力学方程类似于朗之万方程，自然包含了探索-利用的权衡。我们不需要GFlowNet也能获得大部分好处。

**M：** 折中方案是Verbalized Sampling——Google 2025年的工作展示了仅通过prompting策略（不需要训练），就能让LLM的输出多样性提升1.6-2.1倍。这可以作为"穷人版GFlowNet"：在prompt中包含种群分布信息，引导LLM往低密度区域生成变异。成本为零，收益明显。

**P：** 这实际上对应了物理学中的"排斥相互作用"——让新生成的粒子（程序）排斥已有粒子（种群中的程序），从而均匀覆盖状态空间。MAP-Elites已经在做类似的事（通过cell结构保证覆盖），但Verbalized Sampling可以在生成端就施加这个压力。

### 话题四：Ising模型在这个场景中的最佳应用方式是什么？

**P：** HalfCheetah有6个驱动关节。一个自然的Ising映射是把每个关节的控制策略看作一个"自旋"——它可以取多个离散值（不同的控制模式）。关节之间的耦合对应Ising的相互作用 $J_{ij}$。那么系统的能量就是：

$$E = -\sum_{i<j} J_{ij} s_i s_j - \sum_i h_i s_i$$

其中 $s_i$ 是第 $i$ 个关节的控制模式，$J_{ij}$ 编码了关节间的协同效应，$h_i$ 是外场（单关节的最优偏好）。

**M：** 但我不确定这个映射在实践中有多大用处。当前系统中的控制策略是完整的Python函数——它们的变异空间是连续的、高维的、不可简单离散化的。把6个关节映射为Ising自旋太粗糙了。

**P：** 你说得对，直接的Ising映射在控制器设计层面确实太粗糙。但Ising模型的真正价值不在于直接编码控制器，而在于**种群层面的分析工具**。具体来说：

1. **副本对称性破缺（RSB）分析**：如果我们把种群中的程序看作自旋玻璃中的"构型"，那么RSB的破缺模式告诉我们损失景观有多"崎岖"。高度RSB意味着存在大量局部最优，搜索将困难——这时我们应该提高温度或增加岛屿数量。

2. **序参量监控**：定义一个种群的"磁化率" $\chi = \beta(\langle m^2 \rangle - \langle m \rangle^2)$，其中 $m$ 是某个序参量（比如种群中最高适应度的移动平均）。当 $\chi$ 发散时，系统正在经历相变——这是调整策略的最佳时机。

**M：** 这个思路我赞同。实际上，Ising模型最好的应用可能是在**架构选择层面**。当前系统中，5%的变异是"架构变异"（全重写，`architecture_mutation_rate: 0.05`）。这5个架构类型（CPG/NN/StateMachine/Reflexive/Hybrid）可以看作5个"自旋态"，种群在这些态之间的分布就是一个Ising-like系统的热力学态。我们可以监控这个分布来检测架构层面的相变。

**P：** 这其实更接近Potts模型（Ising模型的q态推广）。q=5态Potts模型的相变是一阶的，意味着种群会在某个临界温度突然从多态共存（高多样性）跳到单态主导（mode collapse）。检测这个转变非常重要。

### 话题五：相变检测和利用——这是最被低估的机会吗？

**P：** 是的，我认为这是四份报告中最被低估的洞察。让我具体说明。

扩散模型经历的二阶相变已经被严格证明了。在进化搜索中，类似的相变在以下几个地方发生：

1. **冻结转变**（mode collapse）：当种群多样性下降到某个临界点以下时，系统进入"玻璃态"——所有个体都卡在同一个局部最优附近。这就是mode collapse，对应物理学中的玻璃转变。

2. **错误阈值相变**（Eigen准种模型）：当变异率超过临界值时，选择压力无法维持优良基因型——种群分布崩溃为随机分布。在我们的系统中，LLM temperature过高就会触发这个转变。

3. **对称性破缺**：当种群从均匀探索转变为集中在某个架构/策略上时，发生了对称性破缺。这可以是好的（找到了好策略）也可以是坏的（过早收敛）。

**检测方法非常直接**：计算比热 $C_v = \beta^2 (\langle E^2 \rangle - \langle E \rangle^2)$——这只需要维护适应度的一阶矩和二阶矩（running mean和running variance）。当 $C_v$ 出现峰值时，系统正在经历相变。

**M：** 我完全同意这是最被低估的机会。让我从工程角度补充：当前系统没有任何机制来检测和应对这些转变。`run_evolution()` 中的循环（`island_novelty_parallel.py`第107-437行）是一个简单的迭代循环，没有自适应调整。

但是——我想提出一个比单纯检测更激进的方案：**利用相变来加速搜索**。物理学中，相变点附近的涨落最大，也就是系统探索了最大的状态空间。如果我们能让系统维持在相变点附近（"临界态"），而不是避开相变，那我们就获得了最大的搜索效率。

这就是"自组织临界性"（SOC）的思想。Sakana AI 2026年的Digital Red Queen正是利用了这个原理——通过对抗性进化，系统自发维持在"边缘混沌"状态。

**P：** 你说得对，但实际操控SOC比说起来难得多。更可行的方案是**自适应温度调度**：

- 监控比热 $C_v$ 和磁化率 $\chi$
- 当 $C_v$ 升高时（接近相变），降低温度——利用正在出现的结构
- 当 $C_v$ 降低且多样性也降低时（远离相变且陷入局部最优），提高温度——重新加热以逃离

这比固定退火调度灵活得多，也比SOC容易实现得多。

**M：** 结合AdaEvolve（2026）的层次化自适应结构——局部/全局/统一三层自适应——我们可以实现：
- **局部层**：每个岛的温度根据该岛的 $C_v$ 独立调整
- **全局层**：岛间迁移率根据全局相变信号调整
- **统一层**：LLM的prompt策略根据种群宏观状态调整

这三层对应了不同的时间尺度，自然形成了层次化的自适应系统。

### 话题六：GFlowNet是否是将系综采样引入LLM进化的"终极方案"？

**M：** 理论上，GFlowNet是最优方案。它的训练目标是让生成概率正比于奖励：$\pi_\theta(x) \propto R(x)$，这正是Boltzmann分布。一旦训练完成，每次采样都直接从目标分布抽取——不需要进化循环、不需要种群维护、不需要温度调度。Hu等人（ICLR 2024 Oral）已经证明LLM可以被微调为GFlowNet。

**P：** 但这里有几个严重的实际问题：

1. **奖励函数的高方差**：我们的评估器（HalfCheetah MuJoCo仿真）本身有很大的方差——同一个控制器在不同的随机种子下可以得到差很多的分数。GFlowNet需要稳定的奖励信号来训练。

2. **采样效率**：GFlowNet的训练需要大量的样本和对应的奖励信号。在我们的设置中，每次评估都要跑MuJoCo仿真（耗时约10-120秒），这限制了训练数据的规模。

3. **分布变化**：如果我们在进化过程中修改了评估标准（比如从单纯的速度变为速度+能效），GFlowNet需要重新训练。而基于系综的方法只需要重新定义能量函数。

**M：** 这些都是合理的担忧。让我提出一个分阶段的路线：

**阶段1**（当前可做）：不用GFlowNet，而是用"Amortized Inference"的思想——在prompt中注入种群分布信息，让LLM做隐式的importance sampling。这类似于Verbalized Sampling，零成本。

**阶段2**（中期）：用LoRA微调一个小模型（比如Qwen2.5-Coder 7B），在历史进化数据上训练它按照奖励分布生成代码。这是"轻量级GFlowNet"——不需要在线训练，只用历史离线数据。

**阶段3**（长期）：全功能GFlowNet，用在线RL训练LLM直接按照评估器的奖励分布生成。这需要等到API成本下降、或者本地部署LLM成为可能。

**P：** 这个分阶段路线我同意。但我想指出，即使在阶段3，GFlowNet也不是"终极方案"——因为它本质上是在学习一个**单一温度**的Boltzmann分布。而我们需要的是在**多个温度**下采样（对应不同的探索-利用权衡）。真正的终极方案应该是一个**自适应温度的系综采样器**——它能根据搜索进展动态调整温度。

**M：** 或者说，GFlowNet + 温度参数 = 终极方案。训练一个conditional GFlowNet：$\pi_\theta(x|T)$，给定温度 $T$ 生成分布为 $P(x|T) \propto R(x)^{1/T}$。但是，现在的讨论已经过于理论化了。让我们回到当前能做的事情上来。

### 话题七：实际工程中最大的挑战是什么？

**M：** 我认为最大的挑战是**评估瓶颈**。当前每个程序的评估需要跑MuJoCo仿真，单次评估耗时10-120秒。在300次迭代的标准运行中，总评估时间是几个小时。任何增加评估次数的改进（比如更多的温度层、更频繁的相变检测）都会碰到这个瓶颈。

**P：** 同意。物理学家有个术语叫"critical slowing down"——在相变附近，系统的弛豫时间趋向无穷。在我们的系统中，这意味着在搜索的关键时期，恰恰是最需要多次评估来确认的时候。

我的建议是引入**代理模型**（surrogate model）：用一个简单的神经网络拟合"代码特征 -> 适应度"的映射，在大部分时间用代理模型做快速评估，只在关键节点（比如检测到相变时）使用真实评估器。这在模拟退火文献中叫做"surrogate-assisted optimization"。

**M：** 第二个挑战是**多目标之间的协调**。当前系统有三个岛，三个目标（fitness/novelty_elo/elegance），但它们之间的关系是硬编码的：
- `_sample_from_island_weighted()` 中 `alpha=0.7` 固定了fitness和novelty的权重
- `migration_rate` 是固定的
- `island_competition_rules` 是在配置文件中硬编码的

从系综理论的角度，这应该用自由能 $F = U - TS$ 来统一——$U$ 是适应度，$S$ 是多样性，$T$ 控制权衡。所有三个岛的目标都可以用不同的 $(U, S, T)$ 组合来表达。

**P：** 第三个挑战更根本：**我们不知道搜索空间的结构**。自旋玻璃理论告诉我们，损失景观的"崎岖度"决定了搜索的难度。但在程序空间中，我们对这个景观的拓扑一无所知。是存在少量的全局最优，还是存在大量等价的局部最优？RSB的阶数是多少？

在没有这些信息的情况下，最安全的策略是**保持足够的种群多样性**，使得系统不会被困在任何单一的局部最优中。这正是系综理论的核心教训：ensemble比single instance更鲁棒。

**M：** 第四个挑战是**与现有代码库的兼容性**。当前架构是"冻结上游OpenEvolve + V4子类注入实验层"。任何改进都必须在不修改上游OpenEvolve代码的前提下，通过V4子类的override来实现。这限制了一些底层的修改（比如修改LLM的采样方式）。

**P：** 这实际上是一个优势。物理学中最好的理论是那些不改变底层方程但加入新的对称性/守恒律的理论。我们的改进应该是在V4层添加新的物理约束（温度调度、Metropolis准则、自由能监控），而不是修改底层的进化机制。

## 第二部分：理论框架

### 2.1 统一的数学框架：从配分函数出发

整个policy-evolve系统可以用一个统一的Boltzmann框架描述。设 $\mathcal{X}$ 是所有可能的Python程序的空间（离散的、可数的），定义如下量：

**能量函数**（适应度的负值）：

$$E(x) = -f(x), \quad f(x) = \text{combined\_score}(x)$$

**配分函数**：

$$Z(\beta) = \sum_{x \in \mathcal{X}} \exp(-\beta E(x)) = \sum_{x \in \mathcal{X}} \exp(\beta f(x))$$

**Boltzmann分布**（选择概率）：

$$P(x|\beta) = \frac{\exp(\beta f(x))}{Z(\beta)}$$

**自由能**：

$$F(\beta) = -\frac{1}{\beta} \ln Z(\beta)$$

**热力学量**：

| 物理量 | 公式 | 进化系统含义 |
|--------|------|-------------|
| 内能 $U$ | $\langle E \rangle = -\partial_\beta \ln Z$ | 种群平均适应度（取负） |
| 熵 $S$ | $\beta(U - F) = \beta\langle E \rangle + \ln Z$ | 种群分布的Shannon熵（多样性） |
| 比热 $C_v$ | $\beta^2(\langle E^2\rangle - \langle E\rangle^2)$ | 适应度方差 $\times \beta^2$（对温度的敏感性） |
| 自由能 $F$ | $U - TS$ | 适应度-多样性权衡的最优值 |

**多岛系统的系综描述**：

| 系综类型 | 物理含义 | policy-evolve对应 | 控制量 |
|----------|---------|-------------------|--------|
| 微正则系综 | 固定能量 $E$，所有构型等概率 | 同一MAP-Elites cell内的程序 | 固定cell位置 |
| 正则系综 | 固定温度 $T$，能量涨落 | `_sample_from_elite_cells_temperature` | `sampling_temperature` |
| 巨正则系综 | 粒子数可变 | `_enforce_population_limit` 动态种群 | `population_size` 上限 |
| 多重正则系综 | 多个守恒量 | 多岛系统（不同目标函数） | 各岛的 `competition_rule` |
| 广义Gibbs系综 | 多个有效温度 | $\alpha$-混合采样 | `alpha=0.7` 和各temperature |

### 2.2 核心公式推导

#### 2.2.1 自由适应度公式

种群分布 $\{p_i\}$ 上的自由适应度定义为：

$$\mathcal{F}[\{p_i\}] = \sum_i p_i f_i + T \sum_i p_i \ln p_i$$

第一项是平均适应度 $\langle f \rangle$，第二项是 $-T \cdot S$（负温度乘以熵）。

最小化 $\mathcal{F}$（等价于最大化 $\langle f \rangle - T \cdot (-S)$）得到：

$$p_i^* = \frac{\exp(f_i / T)}{Z}, \quad Z = \sum_j \exp(f_j / T)$$

这正是 `calculate_softmax_heat()` 实现的分布。

#### 2.2.2 Metropolis准则推导

对于岛间迁移，如果程序从岛 $A$（能量函数 $E_A$）迁移到岛 $B$（能量函数 $E_B$），接受概率应为：

$$P_\text{accept} = \min\left(1, \exp\left(-\frac{E_B(x) - E_A(x)}{T_\text{migration}}\right)\right)$$

$= \min\left(1, \exp\left(\frac{f_B(x) - f_A(x)}{T_\text{migration}}\right)\right)$

其中 $f_A(x)$ 和 $f_B(x)$ 分别是程序 $x$ 在岛 $A$ 和岛 $B$ 的适应度评分（`_get_island_score`）。这保证了详细平衡条件。

#### 2.2.3 比热相变检测

定义运行比热：

$$C_v(t) = \frac{\beta^2}{N} \sum_{i=1}^{N} (f_i - \bar{f})^2$$

其中 $N$ 是最近一个窗口内的程序数，$\bar{f}$ 是窗口内的平均适应度。当 $C_v$ 出现尖锐峰值时，系统正在经历相变。

#### 2.2.4 Wang-Landau平坦直方图

对于能量空间 $[E_\min, E_\max]$ 的均匀采样，Wang-Landau算法维护密度估计 $g(E)$，采样概率为 $P(E) \propto 1/g(E)$。在我们的系统中：

1. 将适应度分数离散化到 bins
2. 维护每个 bin 的历史访问计数 $H(E_\text{bin})$
3. 采样时给低访问 bin 的程序更高权重：$w_i \propto 1/H(E_\text{bin}(f_i))$

这保证了种群在整个适应度空间的均匀覆盖。

```python
"""V4 sampling helpers that are not part of upstream openevolve."""

from __future__ import annotations

import math
import numpy as np
from dataclasses import dataclass


@dataclass
class BoltzmannStats:
    """Thermodynamic statistics from a Boltzmann sampling."""
    probabilities: list[float]
    partition_function: float      # Z
    log_partition: float           # ln(Z) (更数值稳定)
    entropy: float                 # S = -sum(p_i * ln(p_i))
    free_energy: float             # F = -T * ln(Z) (= <E> - T*S)
    mean_energy: float             # <E> = sum(p_i * E_i)
    heat_capacity: float           # C_v = beta^2 * Var(E)
    temperature: float             # T


def calculate_softmax_heat(
    scores: list[float], temperature: float = 1.0
) -> list[float]:
    """Calculate a softmax distribution with max-shift stability.

    Backward-compatible: returns only probabilities.
    """
    stats = calculate_boltzmann_ensemble(scores, temperature)
    return stats.probabilities


def calculate_boltzmann_ensemble(
    scores: list[float], temperature: float = 1.0
) -> BoltzmannStats:
    """Full Boltzmann ensemble computation with thermodynamic quantities.

    Interprets scores as negative energies: E_i = -score_i,
    so higher scores = lower energy = more probable.
    """
    if not scores:
        return BoltzmannStats(
            probabilities=[], partition_function=0.0,
            log_partition=float('-inf'), entropy=0.0,
            free_energy=0.0, mean_energy=0.0,
            heat_capacity=0.0, temperature=temperature,
        )

    scores_arr = np.array(scores, dtype=np.float64)
    T = max(temperature, 1e-10)
    beta = 1.0 / T

    # Boltzmann weights with max-shift for numerical stability
    max_score = np.max(scores_arr)
    shifted = (scores_arr - max_score) / T
    exp_scores = np.exp(shifted)
    Z_shifted = np.sum(exp_scores)

    if Z_shifted == 0 or np.isnan(Z_shifted):
        n = len(scores)
        uniform = [1.0 / n] * n
        return BoltzmannStats(
            probabilities=uniform, partition_function=float(n),
            log_partition=math.log(n), entropy=math.log(n),
            free_energy=-T * math.log(n), mean_energy=float(-np.mean(scores_arr)),
            heat_capacity=0.0, temperature=T,
        )

    probs = exp_scores / Z_shifted
    log_Z = math.log(Z_shifted) + max_score / T  # undo the shift

    # Entropy: S = -sum(p_i * ln(p_i))
    log_probs = np.log(np.clip(probs, 1e-300, None))
    entropy = float(-np.sum(probs * log_probs))

    # Mean energy: <E> = -<score> = -sum(p_i * score_i)
    mean_score = float(np.sum(probs * scores_arr))
    mean_energy = -mean_score

    # Heat capacity: C_v = beta^2 * Var(E) = beta^2 * (<E^2> - <E>^2)
    energies = -scores_arr
    mean_E2 = float(np.sum(probs * energies**2))
    var_E = mean_E2 - mean_energy**2
    heat_capacity = beta**2 * max(var_E, 0.0)

    # Free energy: F = -T * ln(Z)
    free_energy = -T * log_Z

    return BoltzmannStats(
        probabilities=probs.tolist(),
        partition_function=float(np.exp(log_Z)) if log_Z < 700 else float('inf'),
        log_partition=log_Z,
        entropy=entropy,
        free_energy=free_energy,
        mean_energy=mean_energy,
        heat_capacity=heat_capacity,
        temperature=T,
    )
```

```python
                        accept = True  # Always accept if better in target
                    else:
                        # Metropolis probability: exp(delta / T)
                        accept_prob = math.exp(
                            delta_score / max(migration_temperature, 1e-10)
                        )
                        accept = random.random() < accept_prob

                    if not accept:
                        logger.debug(
                            "Migration of %s to island %d rejected by Metropolis "
                            "(source_score=%.2f, target_score=%.2f, T=%.1f)",
                            migrant.id[:8], target_island,
                            source_score, target_score, migration_temperature,
                        )
                        continue
                    # --- End Metropolis criterion ---
```

#### P0-3：比热和磁化率的实时监控（相变检测器）

**目标**：在进化循环中实时计算和记录比热 $C_v$，用于检测相变。

**原理**：比热 $C_v = \beta^2 \text{Var}(E)$ 的尖锐峰值标志着相变。在进化搜索中，这对应着种群从"探索态"到"利用态"的转变，或者从一个局部最优跳到另一个的转变。

**新建文件**：`D:\resource\githubrepo\policy-evolve\src\experiment\variants\ensemble_physics.py`

```python
"""Ensemble physics diagnostics for the evolutionary process."""

from __future__ import annotations

import logging
import math
from collections import deque
from dataclasses import dataclass, field

logger = logging.getLogger(__name__)


@dataclass
class PhaseTransitionDetector:
    """Monitors thermodynamic quantities to detect phase transitions.

    Uses a sliding window over recent fitness observations to compute:
    - Heat capacity C_v = beta^2 * Var(f)
    - Susceptibility chi = beta * Var(order_parameter)
    - Population entropy S

    A phase transition is signaled when C_v exceeds a threshold
    (relative to its running average).
    """

    window_size: int = 50
    beta: float = 0.001  # 1/T, default for T=1000
    cv_spike_threshold: float = 3.0  # C_v / mean(C_v) ratio to trigger alert

    # Internal state
    _fitness_window: deque = field(default_factory=lambda: deque(maxlen=50))
    _cv_history: deque = field(default_factory=lambda: deque(maxlen=200))
    _entropy_history: deque = field(default_factory=lambda: deque(maxlen=200))
    _architecture_counts: dict = field(default_factory=dict)
    _phase_transition_log: list = field(default_factory=list)

    def __post_init__(self):
        self._fitness_window = deque(maxlen=self.window_size)
        self._cv_history = deque(maxlen=200)
        self._entropy_history = deque(maxlen=200)

    def observe(self, fitness: float, iteration: int,
                architecture: str | None = None) -> dict:
        """Record a new fitness observation and return diagnostics.

        Returns dict with keys: cv, entropy, phase_transition_detected,
        diagnostics_message.
        """
        self._fitness_window.append(fitness)

        if architecture:
            self._architecture_counts[architecture] = (
                self._architecture_counts.get(architecture, 0) + 1
            )

        result = {
            "cv": 0.0,
            "entropy": 0.0,
            "mean_fitness": fitness,
            "fitness_std": 0.0,
            "phase_transition_detected": False,
            "transition_type": None,
            "diagnostics_message": "",
        }

        if len(self._fitness_window) < 10:
            return result

        # Compute heat capacity
        fitnesses = list(self._fitness_window)
        n = len(fitnesses)
        mean_f = sum(fitnesses) / n
        var_f = sum((f - mean_f) ** 2 for f in fitnesses) / n
        cv = self.beta ** 2 * var_f

        # Compute population entropy from architecture distribution
        total = sum(self._architecture_counts.values()) or 1
        arch_probs = [c / total for c in self._architecture_counts.values()]
        entropy = -sum(p * math.log(p + 1e-10) for p in arch_probs if p > 0)

        self._cv_history.append(cv)
        self._entropy_history.append(entropy)

        result["cv"] = cv
        result["entropy"] = entropy
        result["mean_fitness"] = mean_f
        result["fitness_std"] = math.sqrt(max(var_f, 0))

        # Detect phase transition: C_v spike
        if len(self._cv_history) >= 20:
            mean_cv = sum(self._cv_history) / len(self._cv_history)
            if mean_cv > 0 and cv > self.cv_spike_threshold * mean_cv:
                result["phase_transition_detected"] = True

                # Classify transition type
                recent_entropy = list(self._entropy_history)[-10:]
                earlier_entropy = list(self._entropy_history)[-20:-10]
                if earlier_entropy:
                    entropy_trend = (
                        sum(recent_entropy) / len(recent_entropy)
                        - sum(earlier_entropy) / len(earlier_entropy)
                    )
                    if entropy_trend < -0.1:
                        result["transition_type"] = "freezing"
                        result["diagnostics_message"] = (
                            f"PHASE TRANSITION (freezing): C_v={cv:.4f} "
                            f"(mean={mean_cv:.4f}, ratio={cv/mean_cv:.1f}x), "
                            f"entropy declining. Possible mode collapse. "
                            f"Consider increasing temperature."
                        )
                    elif entropy_trend > 0.1:
                        result["transition_type"] = "melting"
                        result["diagnostics_message"] = (
                            f"PHASE TRANSITION (melting): C_v={cv:.4f} "
                            f"(mean={mean_cv:.4f}, ratio={cv/mean_cv:.1f}x), "
                            f"entropy rising. Population diversifying. "
                            f"Consider decreasing temperature to exploit."
                        )
                    else:
                        result["transition_type"] = "critical"
                        result["diagnostics_message"] = (
                            f"PHASE TRANSITION (critical point): C_v={cv:.4f} "
                            f"(mean={mean_cv:.4f}, ratio={cv/mean_cv:.1f}x). "
                            f"System at criticality — maximum fluctuations."
                        )

                self._phase_transition_log.append({
                    "iteration": iteration,
                    "cv": cv,
                    "mean_cv": mean_cv,
                    "entropy": entropy,
                    "type": result["transition_type"],
                })
                logger.warning(result["diagnostics_message"])

        return result

    def get_recommended_temperature_adjustment(self) -> float:
        """Returns a multiplicative factor for temperature adjustment.

        > 1.0 means increase temperature (more exploration)
        < 1.0 means decrease temperature (more exploitation)
        = 1.0 means no change
        """
        if len(self._cv_history) < 20:
            return 1.0

        recent_cv = list(self._cv_history)[-5:]
        mean_recent_cv = sum(recent_cv) / len(recent_cv)
        overall_mean_cv = sum(self._cv_history) / len(self._cv_history)

        if overall_mean_cv == 0:
            return 1.0

        ratio = mean_recent_cv / overall_mean_cv

        # If C_v is high (phase transition), don't change — system is active
        if ratio > 2.0:
            return 1.0

        # If C_v is low and entropy is low → frozen, increase temperature
        if len(self._entropy_history) >= 10:
            recent_entropy = list(self._entropy_history)[-5:]
            mean_entropy = sum(recent_entropy) / len(recent_entropy)
            max_possible_entropy = math.log(max(len(self._architecture_counts), 2))

            if mean_entropy < 0.3 * max_possible_entropy and ratio < 0.5:
                return 1.5  # Increase temperature by 50%

        # If C_v is low and entropy is high → well-mixed, decrease temperature
        if len(self._entropy_history) >= 10:
            if mean_entropy > 0.8 * max_possible_entropy and ratio < 0.5:
                return 0.8  # Decrease temperature by 20%

        return 1.0

    def get_summary(self) -> dict:
        """Return summary statistics."""
        return {
            "total_observations": len(self._fitness_window),
            "transitions_detected": len(self._phase_transition_log),
            "current_cv": self._cv_history[-1] if self._cv_history else 0.0,
            "current_entropy": self._entropy_history[-1] if self._entropy_history else 0.0,
            "architecture_distribution": dict(self._architecture_counts),
            "transition_log": list(self._phase_transition_log),
        }
```

```python
        If island_idx is given, uses that island's competition rule to set weights.
        If weights are explicitly given, uses those instead.

            if elo is not None:
                novelty = float(elo) / 1000.0  # Normalize ELO to ~[0, 2]

            elegance = float(program.metrics.get("elegance_total", 0.0) or 0.0)

        # Energy is negative of weighted score (lower energy = better)
        score = (
            weights.get("fitness", 0.0) * fitness
            + weights.get("novelty", 0.0) * novelty
            + weights.get("elegance", 0.0) * elegance
        )
        return -score

        against population diversity (entropy).

            if pid in self.programs

        mean_E = sum(energies) / len(energies)

        scores = [-e for e in energies]
        from src.experiment.variants.island_novelty_sampling import (
            calculate_boltzmann_ensemble,
        )
        stats = calculate_boltzmann_ensemble(scores, temperature=T)

        return mean_E - T * stats.entropy
```

#### P1-2：副本交换Monte Carlo（Replica Exchange）

**目标**：让不同岛在不同温度下运行，并周期性地尝试交换构型。

**原理**：副本交换Monte Carlo（REMC）让多个副本在不同温度下并行模拟，然后尝试交换相邻温度的构型。交换准则是：

$$P_\text{swap}(i \leftrightarrow j) = \min\left(1, \exp\left((\beta_i - \beta_j)(E_i - E_j)\right)\right)$$

这允许高温副本帮助低温副本翻越能垒。

```python
    def replica_exchange(self) -> list[tuple[int, int, bool]]:
        """Attempt replica exchange between adjacent islands.

        Each island has an effective temperature derived from its
        competition rule and sampling behavior. Exchanges are attempted
        between adjacent islands using the Metropolis criterion.

        Returns list of (island_i, island_j, accepted) tuples.
        """
        if len(self.islands) < 2:
            return []

        # Define effective temperatures for each island
        # Fitness island: low T (exploitative), Novelty island: high T (explorative)
        island_temperatures = {}
        for idx in range(len(self.islands)):
            rules = self._v4("island_competition_rules", {})
            default_mode = self._v4("island_competition_rule", "fitness")
            rule = rules.get(idx, rules.get(str(idx), default_mode))
            if rule == "fitness":
                island_temperatures[idx] = 100.0  # Low T
            elif rule == "novelty_elo":
                island_temperatures[idx] = 1000.0  # High T
            elif rule == "elegance":
                island_temperatures[idx] = 500.0   # Medium T
            else:
                island_temperatures[idx] = 500.0

        exchanges = []
        island_indices = sorted(island_temperatures.keys())

        for i in range(len(island_indices) - 1):
            idx_i = island_indices[i]
            idx_j = island_indices[i + 1]

            # Get representative programs (best from each island)
            best_i = self._get_island_best(idx_i)
            best_j = self._get_island_best(idx_j)

            if best_i is None or best_j is None:
                exchanges.append((idx_i, idx_j, False))
                continue

            T_i = island_temperatures[idx_i]
            T_j = island_temperatures[idx_j]
            beta_i = 1.0 / max(T_i, 1e-10)
            beta_j = 1.0 / max(T_j, 1e-10)

            # Use unified energy for comparison
            E_i = self.compute_energy(best_i, idx_i)
            E_j = self.compute_energy(best_j, idx_j)

            # Replica exchange criterion
            delta = (beta_i - beta_j) * (E_i - E_j)
            accept = delta >= 0 or random.random() < math.exp(delta)

            if accept:
                # Swap: copy best of island_i to island_j and vice versa
                import uuid
                for src, dst, prog in [
                    (idx_i, idx_j, best_i),
                    (idx_j, idx_i, best_j)
                ]:
                    copy = Program(
                        id=str(uuid.uuid4()),
                        code=prog.code,
                        changes_description=prog.changes_description,
                        language=prog.language,
                        parent_id=prog.id,
                        generation=prog.generation,
                        metrics=prog.metrics.copy() if prog.metrics else {},
                        metadata={
                            **prog.metadata,
                            "island": dst,
                            "replica_exchange": True,
                            "source_island": src,
                        },
                    )
                    self.add(copy, target_island=dst)

                logger.info(
                    "Replica exchange accepted: island %d <-> %d "
                    "(T=%.0f/%.0f, E=%.2f/%.2f, delta=%.3f)",
                    idx_i, idx_j, T_i, T_j, E_i, E_j, delta,
                )

            exchanges.append((idx_i, idx_j, accept))

        return exchanges

    def _get_island_best(self, island_idx: int) -> Program | None:
        """Get the best program in an island by that island's score."""
        island_programs = [
            self.programs[pid] for pid in self.islands[island_idx]
            if pid in self.programs
        ]
        if not island_programs:
            return None
        return max(
            island_programs,
            key=lambda p: self._get_island_score(p, island_idx),
        )
```

#### P1-4：Verbalized Sampling（Prompt级多样性注入）

```python
    def _get_population_distribution_summary(self) -> str:
        """Generate a summary of current population for verbalized sampling."""
        arch_counts = {}
        total = 0
        for pid, prog in self.database.programs.items():
            arch = prog.metadata.get("architecture_type", "unknown")
            # Also check docstring label
            if arch == "unknown" and prog.code:
                for label in ["CPG", "NeuralNet", "StateMachine", "Reflexive", "Hybrid"]:
                    if label.lower() in prog.code.lower():
                        arch = label
                        break
            arch_counts[arch] = arch_counts.get(arch, 0) + 1
            total += 1

        if total == 0:
            return ""

        lines = ["Current population distribution:"]
        for arch, count in sorted(arch_counts.items(), key=lambda x: -x[1]):
            pct = count / total * 100
            lines.append(f"  - {arch}: {count} ({pct:.0f}%)")

        # Identify underrepresented architectures
        all_archs = {"CPG", "NeuralNet", "StateMachine", "Reflexive", "Hybrid"}
        missing = all_archs - set(arch_counts.keys())
        if missing:
            lines.append(f"  UNDERREPRESENTED: {', '.join(missing)}")
            lines.append("Consider trying an underrepresented architecture for novelty bonus.")

        return "\n".join(lines)
```

#### P2-2：Wang-Landau平坦直方图采样

**目标**：实现Wang-Landau算法，确保种群在整个适应度空间的均匀覆盖。

**原理**：Wang-Landau算法迭代估计状态密度 $g(E)$，采样概率为 $P(E) \propto 1/g(E)$，从而实现能量空间的均匀采样。这解决了标准Boltzmann采样在高温时过度探索低能态、低温时过度聚焦高能态的问题。

**修改文件**：`D:\resource\githubrepo\policy-evolve\src\experiment\variants\island_novelty_sampling.py`

**新增函数**：

```python
class WangLandauSampler:
    """Wang-Landau flat histogram sampler for uniform energy coverage.

    Maintains an estimate of the density of states g(E) and samples
    with probability 1/g(E) to ensure flat histogram in energy space.
    """

    def __init__(self, num_bins: int = 20, initial_f: float = 1.0,
                 flatness_criterion: float = 0.8, f_min: float = 1e-8):
        self.num_bins = num_bins
        self.log_g = np.zeros(num_bins)  # log(g(E)) for each energy bin
        self.histogram = np.zeros(num_bins, dtype=int)
        self.f = initial_f  # modification factor (starts at e^1)
        self.flatness_criterion = flatness_criterion
        self.f_min = f_min
        self.e_min = None
        self.e_max = None

    def update_energy_range(self, energies: list[float]):
        """Update the energy range based on observed energies."""
        if not energies:
            return
        if self.e_min is None:
            self.e_min = min(energies) - 1.0
            self.e_max = max(energies) + 1.0
        else:
            self.e_min = min(self.e_min, min(energies) - 1.0)
            self.e_max = max(self.e_max, max(energies) + 1.0)

    def energy_to_bin(self, energy: float) -> int:
        if self.e_min is None or self.e_max is None:
            return 0
        if self.e_max == self.e_min:
            return 0
        bin_idx = int((energy - self.e_min) / (self.e_max - self.e_min) * self.num_bins)
        return max(0, min(self.num_bins - 1, bin_idx))

    def get_sampling_weights(self, energies: list[float]) -> list[float]:
        """Return sampling weights inversely proportional to g(E).

        Programs in under-visited energy regions get higher weight.
        """
        if not energies or self.e_min is None:
            return [1.0 / len(energies)] * len(energies) if energies else []

        weights = []
        for e in energies:
            bin_idx = self.energy_to_bin(e)
            # Weight inversely proportional to density of states
            w = np.exp(-self.log_g[bin_idx])
            weights.append(w)

        total = sum(weights)
        if total > 0:
            return [w / total for w in weights]
        return [1.0 / len(energies)] * len(energies)

    def record_visit(self, energy: float):
        """Record a visit to an energy state."""
        bin_idx = self.energy_to_bin(energy)
        self.log_g[bin_idx] += self.f
        self.histogram[bin_idx] += 1

        # Check flatness and update modification factor
        if self._is_flat():
            self.f *= 0.5  # Reduce modification factor
            self.histogram[:] = 0  # Reset histogram

    def _is_flat(self) -> bool:
        """Check if histogram is sufficiently flat."""
        nonzero = self.histogram[self.histogram > 0]
        if len(nonzero) < self.num_bins * 0.5:
            return False
        mean_h = np.mean(nonzero)
        return np.all(nonzero >= self.flatness_criterion * mean_h)
```
