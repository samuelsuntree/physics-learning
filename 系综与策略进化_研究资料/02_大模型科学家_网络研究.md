# LLM驱动的策略进化与统计物理系综理论框架——大模型科学家研究报告

## 一、LLM驱动的代码/策略进化

### 1.1 FunSearch (DeepMind, 2023)
- 分布式系统：程序数据库 + 采样器 + 评估器
- 岛屿模型：m个独立岛屿，每4小时丢弃m/2个最差岛屿
- 岛屿内部按签名(signature)分簇
- Best-Shot Prompting: 从数据库采样k个程序，按分数排序

### 1.2 Evolution through Large Models (ELM, Lehman et al., 2022)
- Diff模型在GitHub diff上训练
- 与MAP-Elites结合，生成数十万个功能性Python程序
- LLM与进化协同工作，可以将彼此引导到正确的解空间区域

### 1.3 AlphaEvolve (DeepMind, 2025)
- Gemini 2.0 Flash（广度）+ Gemini 2.0 Pro（深度）双模型架构
- 发现了48次标量乘法做4x4复数矩阵乘法的算法
- 优化Google数据中心任务调度，回收约0.7%全球计算资源

### 1.4 开源生态
- OpenELM (CarperAI): 基于Langchain，支持任意LLM API
- CodeEvolve (Inter.co): 岛屿遗传算法 + 加权LLM集成
- GigaEvo (FusionBrain Lab): Redis + DAG + 多岛屿MAP-Elites
- SkyDiscover (UC Berkeley): 模块化四组件架构，200+任务

### 1.5 AdaEvolve (2026)
- 将LLM驱动的进化重新表述为层次化自适应优化问题
- 三层自适应结构：局部适应 / 全局适应 / 统一决策

### 1.6 Digital Red Queen (Sakana AI, 2026)
- 对抗性开放式进化，在Core War中LLM迭代进化汇编程序
- 随轮次增加，warriors变得越来越通用

## 二、LLM采样与统计物理的数学联系

### 2.1 Softmax与Boltzmann分布的同构性
- logit z_i 对应 -E_i（负能量）
- LLM temperature T 对应热力学温度
- softmax归一化常数对应配分函数Z
- 低温(T→0): 贪心解码；高温(T→∞): 随机采样

### 2.2 自回归语言模型本质上是能量基模型
- ARM和EBM之间的显式双射(Blondel et al.)
- 该对应是soft Bellman方程的特殊情况
- 揭示了ARM具有前瞻规划能力的原因

### 2.3 Boltzmann-GPT (2026)
- "嘴巴不是大脑"架构：DBM + 适配器 + 冻结GPT-2
- DBM能量函数区分连贯/不连贯配置

### 2.4 LLM与热力学的深层联系
- Shannon信息熵与Boltzmann熵公式形式相同
- 配分函数、内能、自由能等概念引入LLM分析框架

## 三、多样性维护与Mode Collapse的解决方案

### 3.1 MAP-Elites + LLM结合的最新进展
- ELM + MAP-Elites (2022): LLM变异 + QD搜索
- QDAIF (ICLR 2024): LLM同时做变异+适应度+多样性评估
- CycleQD (ICLR 2025): SVD变异 + 模型合并交叉
- GigaEvo (2025): 多岛屿MAP-Elites + 周期迁移

### 3.2 Verbalized Sampling (Zhang et al., 2025)
- 无需训练的prompting策略
- 多样性提升1.6-2.1x，不牺牲准确性

### 3.3 从统计物理角度理解Mode Collapse
- 玻璃态转变而非平衡相变
- 解决方案：并行退火 / 种群退火 / 岛屿模型

## 四、能量基模型在代码生成中的应用

### 4.1 Energy-Based Diffusion Language Models (NVIDIA)
- 在每个扩散步骤引入EBM
- 接近自回归模型性能，采样速度提升1.3x

### 4.2 GFlowNet
- 采样概率正比于奖励，等价于Boltzmann采样
- "Amortizing intractable inference in LLMs" (ICLR 2024 Oral): 微调LLM作为GFlowNet

### 4.3 奖励引导的解码
- GenARM (ICLR 2025): next-token奖励预测
- Reward-Guided Speculative Decoding (RSD)

## 五、可操作的算法框架

### 5.1 Boltzmann采样引入LLM代码进化
- 定义能量函数 E(c) = -α·f_perf(c) + β·f_complex(c) + γ·f_novelty(c)
- 用Boltzmann分布替代锦标赛选择
- LLM temperature控制变异的"热力学温度"

### 5.2 退火调度设计
- 方案A: 经典指数退火 T(t) = T_0·α^t
- 方案B: 并行退火(Parallel Tempering) + Metropolis交换
- 方案C: 自适应退火(AdaEvolve启发)

### 5.3 系综理论完整映射表
| 统计物理 | LLM进化系统 |
|---------|------------|
| 微正则系综(NVE) | 固定种群+固定能量预算+固定代码长度 |
| 正则系综(NVT) | 固定种群+固定温度+可变适应度 |
| 巨正则系综(μVT) | 可变种群+固定温度+化学势控制 |
| 配分函数Z | 所有程序的适应度加权求和 |
| 自由能F=-kTlnZ | 适应度-多样性的最优平衡点 |
| F=U-TS | 进化目标=平均性能-T×多样性 |

### 5.4 分阶段实施计划
- Phase 1: 复现FunSearch架构
- Phase 2: 引入系综框架（核心创新）
- Phase 3: 自适应与相变检测
- Phase 4: GFlowNet集成
