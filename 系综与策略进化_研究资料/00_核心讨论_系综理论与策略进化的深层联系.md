## 引言：为什么一个代码进化系统看起来像一个统计力学系统？

当我们第一次打开 policy-evolve 的代码——一个用大语言模型来进化 MuJoCo 机器人控制策略的系统——我们发现了一些令人惊讶的东西：这个系统的工程师们，也许并非有意地，构建了一个与统计力学在数学结构上深度同构的系统。

`calculate_softmax_heat()` 函数就是 Boltzmann 分布。ELO 评分系统的胜率公式就是 Fermi-Dirac 分布。多岛屿模型就是 parallel tempering 的天然载体。配分函数被计算了——然后被丢弃了。

这不是巧合。这篇文档试图解释为什么会这样，以及这种同构性意味着什么。

## 第一章：理解 policy-evolve 系统——从物理学家的眼睛看过去

### 1.1 这个系统在做什么？

policy-evolve 的核心循环可以用一句话概括：**在程序空间中寻找一个函数 `compute_action(observation) -> action`，使得 HalfCheetah 机器人跑得又快又省力。**

具体来说：
- **状态空间**：所有合法的 Python 函数（一个离散的、组合爆炸的空间）
- **能量函数**：E(code) = -(forward_velocity - 0.1 × ctrl_cost²)——负的 MuJoCo 奖励
- **动力学**：LLM 生成代码变异（diff 或全重写），就像 Monte Carlo 中的试探性移动
- **选择**：基于适应度的 softmax 采样，决定哪些"状态"存活

从物理学家的视角看，这就是一个**在极高维离散空间上的蒙特卡洛采样过程**，目标是找到能量函数的（近似）全局最小值。

### 1.2 双层架构的物理意义

policy-evolve 的"冻结上游 + 子类注入"架构有一个有趣的物理类比：

- **上游层**（OpenEvolve）定义了"基本物理规律"——MAP-Elites 网格结构、种群管理、prompt 构建规则
- **实验层**（V4 variants）定义了"具体系统的哈密顿量"——岛屿竞争规则、新颖性 ELO、架构变异率

这就像统计力学中，你有一套通用的数学框架（配分函数、自由能、响应函数），但具体的物理由哈密顿量 H 的选择决定。policy-evolve 的上游层是框架，实验层是 H。

### 1.3 种群作为系综

在统计物理中，**系综**是同一系统在相同宏观约束下所有可能微观状态的集合，每个状态被赋予一个概率权重。

policy-evolve 的种群（`ProgramDatabase` 中的所有 `Program` 对象）就是一个系综：
- 每个 `Program`（一段 Python 代码）= 一个微观状态
- `combined_score`（适应度）= 负能量 -E
- `sampling_temperature`（当前=1000）= 温度 T
- MAP-Elites 网格 = 相空间的粗粒化

但这里有一个关键的区别：物理系统的系综在热平衡下是**稳态的**，分布不随时间变化；而 policy-evolve 的种群是**非平衡的**，它在不断进化。这引出了一个深刻的问题：**进化系统能否达到某种意义上的"热平衡"？如果能，那个平衡态是什么？**

答案可能是：当进化收敛时（early stopping 被触发），种群的适应度分布趋向稳态——这就是进化系统的"热平衡"。而 `sampling_temperature` 控制了这个平衡态的"宽度"：高温（T=1000，当前配置）→ 宽分布（高多样性），低温 → 窄分布（收敛到少数精英）。

### 1.4 初始程序：一个有趣的"基态"

初始程序是一个简单的线性策略 `tanh(W @ obs + b)`（`initial_program.py`），其中 W 是手工设计的权重矩阵。这是进化的起点。

在物理类比中，这就像是一个**已知的低能态**（但不一定是基态）。进化过程从这里出发，通过 LLM 变异探索邻近的构型空间，试图找到更低的能量态。

有趣的是，初始程序的 W 矩阵编码了 HalfCheetah 的一些先验知识（前后腿的协调模式）。这些先验在进化中扮演什么角色？从物理角度看，它们就像**外部磁场**——偏置系统朝特定方向对齐。进化是否能"突破"这些先验，找到完全不同的策略？5% 的架构变异率就是为此设计的——一种"量子隧穿"机制，允许系统跃迁到完全不同的能量盆地。

## 第二章：平衡态统计物理的映射——不只是类比

### 2.1 已经存在的精确数学对应

让我们不加比喻地、精确地列出 policy-evolve 中已经存在的统计物理结构：

**（a）Boltzmann 分布**

`island_novelty_sampling.py` 第 8-21 行：
```python
def calculate_softmax_heat(scores, temperature=1.0):
    shifted_scores = scores_arr - np.max(scores_arr)
    exp_scores = np.exp(shifted_scores / max(temperature, 1e-6))
    sum_exp = np.sum(exp_scores)  # ← 这就是配分函数 Z
    return (exp_scores / sum_exp).tolist()
```

这严格等于：P(i) = exp(s_i / T) / Z，即 Boltzmann 分布，其中 s_i = -E_i（负能量等于适应度分数）。

**（b）Fermi-Dirac 分布**

`novelty_elo.py` 第 30-32 行：
```python
def expected_score(elo_a, elo_b):
    return 1.0 / (1.0 + 10.0 ** ((elo_b - elo_a) / 400.0))
```

这等价于 Fermi-Dirac 分布 f(ε) = 1/(1 + exp(β(ε - μ)))，其中 β = ln(10)/400 ≈ 0.00576，相当于"有效温度" kT = 400/ln(10) ≈ 173.7 ELO 分。

在 ELO 系统中，每个程序的 ELO 分就像费米子的能级。胜率公式告诉我们：两个程序的"能级差"越大，胜率越确定——这和费米子在有限温度下的占据概率是完全相同的数学。

**（c）广义 Gibbs 系综**

`island_novelty_database.py` 第 488-497 行的双系综混合：
```python
alpha = 0.7
weights = [alpha * r_heat + (1-alpha) * n_heat for r_heat, n_heat in zip(reward_heats, novelty_heats)]
```

这是两个正则系综的线性组合：P = α·P_reward + (1-α)·P_novelty。在物理中，这对应于广义 Gibbs 系综（GGE），其中系统受多个守恒量约束。这里的两个"守恒量"是 reward 和 novelty——两个不同的"好"的维度。

**（d）Parallel Tempering 的天然载体**

多岛系统（Island 0: fitness，Island 1: novelty_elo）在物理上就是在不同"哈密顿量"下演化的系统副本。当前的迁移机制（`migrate_programs()`，第 614-693 行）是确定性地复制 top programs——这是 parallel tempering 的简化版本（完整版本需要 Metropolis 接受准则）。

### 2.2 缺失但重要的物理量

**（a）配分函数 Z 的信息被丢弃**

`calculate_softmax_heat()` 中的 `sum_exp` 就是配分函数 Z，但只被用于归一化后丢弃。在统计物理中，Z 是最重要的量——自由能 F = -kT ln Z 编码了系统的全部热力学信息。

如果我们追踪 Z 随进化代数的变化，可以得到：
- Z 增大 → 搜索空间中"好的解"变多了（温度有效降低，或能量景观变平坦了）
- Z 减小 → 好的解变少了（搜索可能陷入了死胡同）
- Z 的对数导数 = 平均能量（`<E> = -∂ln Z/∂β`）——这给出种群的平均适应度

**（b）自由能 F = U - TS**

进化系统的"自由能"概念（Prügel-Bennett, PRL 1994）：
$$F_{\text{evolve}} = \langle f \rangle - T \cdot S_{\text{pop}}$$

其中 <f> 是种群平均适应度，S_pop 是种群多样性（Shannon 熵），T 是有效温度。

这个公式说的是：**一个好的进化系统不应该只追求高适应度（低能量），也不应该只追求高多样性（高熵），而是应该最小化自由能——在给定温度下，找到适应度和多样性的最优平衡。**

当前 policy-evolve 隐式地在做这件事（MAP-Elites 维护多样性，选择压力推动适应度），但没有显式地计算和优化自由能。

**（c）比热容和响应函数**

比热容 C_v = Var(E) / (kT)² 描述了系统对温度变化的响应。如果 C_v 突然升高，说明系统处于**相变**附近——这是搜索动态发生质变的信号。

当前代码不计算任何此类诊断量。

### 2.3 一个深刻的问题：LLM 内部的能量景观 vs 评估器的能量景观

这里存在一个物理学家必须面对的微妙问题。系统中其实有**两个能量景观**：

1. **LLM 的隐式能量景观**：DeepSeek V3 的训练使它"倾向于"生成某些风格的代码。当 temperature=0.8 时，它是在这个隐式能量面上做 Boltzmann 采样。这个能量面反映的是"什么是好的 Python 代码"的通用知识。

2. **评估器的能量景观**：MuJoCo 仿真给出的 combined_score 定义了另一个能量面。这个面反映的是"什么代码能让 HalfCheetah 跑得好"。

这两个能量面在大部分区域是不对齐的——好的 Python 代码不一定能控制好机器人。进化过程的本质，就是**利用第一个能量面上的采样来探索第二个能量面上的低能态**。

这有点像**重要性采样（importance sampling）**：我们用一个容易采样的分布（LLM 的输出分布）来近似一个难以直接采样的目标分布（高适应度代码的分布）。选择机制起到了重要性权重的作用。

从这个角度看，进化的效率取决于两个能量面之间的"重叠度"——如果 LLM 倾向于生成的代码类型中恰好包含很多高适应度的候选，搜索就会很高效；反之就会很低效。这也解释了为什么 prompt 工程如此重要——好的 prompt 本质上是在调整 LLM 的能量面，使其更好地"覆盖"适应度景观的低能区域。

### 2.4 非平衡性：进化不是热平衡

必须诚实地承认：policy-evolve 不是一个平衡态系统。它有几个关键的非平衡特征：

1. **种群在持续变化**：新程序不断被生成和淘汰，分布不断演变
2. **"能量函数"不是静态的**：Novelty ELO 的基线会更新，种群组成的变化会改变新颖性度量的含义
3. **没有详细平衡**：LLM 的变异算子不满足 detailed balance——从代码 A 变异到代码 B 的概率，不等于从 B 变异到 A 的概率乘以 Boltzmann 因子

但这并不意味着平衡态统计物理的概念就不适用了。原因有二：

第一，**非平衡系统可以在长时间尺度上近似平衡**。当进化接近收敛时，种群的宏观性质（平均适应度、多样性分布）趋向稳态。在这个时间尺度上，平衡态的概念（自由能、熵、相变）是良好的近似。

第二，更重要的是，**许多非平衡系统的有效描述仍然使用平衡态的数学工具**。就像流体力学用平衡态的状态方程来闭合方程组一样，进化系统的宏观动力学可以用准平衡态的自由能景观来描述。

## 第三章：跨学科的联想与思想碰撞

### 3.1 对话：LLM 本身就是一个 Boltzmann 机

**物理学家(P)：** 2024年底 Blondel 等人的论文证明了一个惊人的事实：自回归语言模型（ARM）和能量基模型（EBM）之间存在精确的数学双射。也就是说，当 DeepSeek V3 在 temperature=0.8 下生成代码时，它实际上是在一个隐式定义的能量景观上做 Boltzmann 采样——β = 1/0.8 = 1.25。

**大模型科学家(M)：** 这个等价性是通过 soft Bellman 方程建立的。在最大熵强化学习的框架下，自回归生成过程等价于在序列空间上定义一个全局的能量函数，然后做 Boltzmann 采样。这解释了 LLM 的"前瞻能力"——因为 EBM 的定义天然是全局的，它"看到"的是整个序列的能量，而不只是下一个 token。

**P：** 所以 policy-evolve 中发生了一件有趣的事：一个 Boltzmann 机（LLM）在为另一个 Boltzmann 系统（进化种群）生成候选状态。这是两层 Boltzmann 采样的嵌套。

**M：** 对。而且 Boltzmann-GPT（2026年1月）的工作走得更远——它用一个显式的深度 Boltzmann 机（DBM）作为"世界模型"，然后用一个冻结的 GPT 来"朗读"DBM 的信念状态。"嘴巴不是大脑"——语言模型只是一个解码器，真正的结构化知识在 Boltzmann 机里。

**P：** 这让我想到一个联想：如果我们把 policy-evolve 的评估器（MuJoCo 仿真）看作一个"世界模型"，它定义了控制策略空间上的能量景观。而 LLM 是"嘴巴"——它把这个能量景观上的信息"朗读"成代码。进化过程就是在训练这个"嘴巴"更好地表达"大脑"（评估器）知道的东西。

**M：** 但有个本质差异——Boltzmann-GPT 中，DBM 和 GPT 是通过一个可学习的 adapter 连接的，端到端训练。而 policy-evolve 中，LLM 和评估器之间的连接是通过**prompt 工程**和**选择压力**建立的——这是一种更弱、更间接的耦合。

**P：** 更弱，但也更灵活。物理系统中的耦合常数是固定的；prompt 可以随时调整。而且选择压力是一种**非线性反馈**——适应度高的代码被更多地采样为 prompt 的一部分，这改变了 LLM 的条件分布。从动力学角度看，这是一个**自组织系统**。

### 3.2 对话：Ising 模型在哪里？

**M：** 你能解释一下 Ising 模型在这个场景中最自然的映射吗？代码空间看起来不像是自旋格点。

**P：** 最直接的映射不是代码本身，而是**代码控制的物理系统**——HalfCheetah。它有 6 个关节，每个关节在任意时刻可以施加正或负的扭矩。如果我们把扭矩的符号看作自旋（+1 或 -1），6 个关节就是 6 个自旋。相邻关节之间有物理耦合（前后腿需要协调），这就是自旋间的耦合常数 J_ij。

一个"好的步态"对应于特定的自旋关联模式。比如，跑步时前后腿反相位运动——这是反铁磁耦合。而同一条腿的上下关节同相位运动——这是铁磁耦合。

**M：** 所以进化出一个好的策略，就是找到一组 J_ij，使得 Ising 系统的基态对应于好的步态？

**P：** 差不多，但更精确的说法是：进化在搜索的不是耦合常数 J_ij，而是**整个动力学规则**——`compute_action()` 函数定义了自旋之间的相互作用如何依赖于当前状态。这比 Ising 模型复杂得多，因为 Ising 模型的 J_ij 是常数，而这里的"耦合"是状态依赖的。

**M：** 但 Ising 模型的思想框架仍然有用？

**P：** 非常有用，至少在两个层面：

第一，**序参量的选择**。Ising 模型中，磁化强度 m = <s_i> 是最自然的序参量——它区分了有序相（所有自旋对齐）和无序相（自旋随机）。在 policy-evolve 中，我们可以定义类似的序参量：比如"平均关节协调度"——所有策略的关节扭矩符号关联的平均值。如果这个量出现突变，说明种群正在经历从"无序策略"到"有序步态"的相变。

第二，**临界现象**。在 Ising 模型的临界温度附近，系统展现出长程关联——一个自旋的翻转可以引发远处自旋的连锁反应。在进化中，如果系统处于"临界点"附近，一个小的代码变异可能引发策略行为的质变。这通常是最有意义的搜索区域——既不是高温的完全随机（无结构的探索），也不是低温的完全冻结（无意义的利用）。

### 3.3 对话：相变是最被低估的机会

**P：** 我认为相变检测是整个统计物理框架中最具实用价值但最被忽视的部分。

当前系统中没有任何对"搜索动态"的诊断。你不知道种群是处于"高温无序相"（变异太强，好的解存活不了）还是"低温冻结相"（种群多样性丧失，陷入局部最优），还是"临界相"（有趣的事情正在发生）。

**M：** 那具体表现为什么？怎么检测？

**P：** 最简单的是监控"比热容"：C_v ∝ Var(fitness) / T²。如果你画出 C_v 随进化代数的曲线，在相变点附近你会看到一个尖峰——种群适应度的方差急剧增大。这意味着种群中同时存在高适应度和低适应度的个体，系统正在经历结构性的重组。

另一个信号是"关联长度"的发散。在代码空间中，这可以用"编辑距离的关联函数"来衡量——如果高适应度的程序之间的编辑距离突然变大或变小，说明搜索正在经历相变。

**M：** 这让我想到 FunSearch 的一个有趣机制——每 4 小时丢弃一半最差岛屿。这在物理上相当于一次"淬火"（quench）——突然改变系统条件。这个操作可能正好是在人为制造"相变点"附近的条件。

**P：** 精确的观察！而且 Digital Red Queen（Sakana AI, 2026）的对抗性进化本质上是在不断移动能量景观——每一轮的"对手"变了，你的能量面就变了。这就像一个被外部周期性驱动的系统——非平衡统计物理中研究了很多这样的系统，它们可以展现出平衡系统没有的相（比如时间晶体）。

### 3.4 对话：GFlowNet——从 Boltzmann 采样到因果推断

**M：** 让我提出一个可能更深远的联系。GFlowNet（Bengio et al., 2021）实现了一件事：给定一个奖励函数 R(x)，训练一个生成模型使得采样概率 π(x) ∝ R(x)。这就是在 T=1 下的精确 Boltzmann 采样。

2024 年 ICLR Oral 论文 "Amortizing intractable inference in LLMs" 更进一步：微调 LLM 作为 GFlowNet，使它能从难以处理的后验分布中采样。

**P：** 这太有意思了。你是说，理论上我们可以训练一个 LLM，使得它生成代码的概率恰好正比于代码的适应度？

**M：** 正是如此。当前系统中，LLM 的生成分布和适应度景观是分离的——LLM 生成它"觉得好"的代码，然后进化过程通过选择来对齐。GFlowNet 的思路是**直接训练 LLM 来按适应度采样**，跳过进化选择这个中间步骤。

**P：** 从物理角度看，这就像是从 MCMC（Markov Chain Monte Carlo）升级到了完美的 Boltzmann 采样器。MCMC 需要漫长的马尔可夫链才能达到平衡，而 GFlowNet 是**摊销化的（amortized）**——一旦训练完成，每次采样都是独立的、精确的。

**M：** 但现实中的挑战是：训练 GFlowNet 本身需要大量的能量函数评估（MuJoCo 仿真）。所以这更适合作为 Phase 4 的长期目标——先用传统的进化 + 统计物理框架把系统做好，积累训练数据，然后训练 GFlowNet 来加速。

### 3.5 对话：自由能原理与进化的目标函数

**P：** 让我提出一个更抽象但可能更有深度的联系。Karl Friston 的自由能原理（Free Energy Principle）认为，生物系统（从单细胞到大脑）的行为可以统一描述为"最小化变分自由能"。自由能 F = U - TS 的最小化同时追求低能量（准确预测/高适应度）和高熵（灵活性/鲁棒性）。

进化本身——不管是生物进化还是 policy-evolve 中的代码进化——也可以理解为一个最小化自由能的过程：
$$F_{\text{evolve}} = -\langle \text{fitness} \rangle + T \cdot S_{\text{pop}}$$

（注意符号：我们最大化适应度 = 最小化负适应度；加上温度乘以熵项是因为我们需要保持多样性来避免过早收敛。）

**M：** 这和 Soft Actor-Critic (SAC) 的目标函数一模一样！SAC 的策略优化目标是：
$$\pi^* = \arg\max_\pi \sum_t \mathbb{E}[r(s_t, a_t) + \alpha \mathcal{H}(\pi(\cdot|s_t))]$$

其中 α 是温度参数，H 是策略熵。Blondel 等人证明的 ARM-EBM 等价性正好是 soft Bellman 方程的特殊情况。

所以我们有了一个贯穿三个领域的统一原理：
- **统计物理**：最小化自由能 F = U - TS
- **强化学习**：最大化 reward + α·entropy（最大熵RL）
- **进化搜索**：最大化 fitness - T·diversity_cost

这三个目标函数在数学上是完全等价的。温度 T（或 α）是唯一的自由参数，它控制了"确定性利用"和"随机探索"之间的平衡。

**P：** 而且这个平衡不应该是静态的——这是模拟退火教给我们的最重要的一课。最优策略是**动态地调整温度**：开始时高温（广泛探索），然后逐渐降温（聚焦利用），在检测到"有趣的相变"时暂停降温甚至短暂升温（深入探索新发现的结构）。

### 3.6 联想：从自旋玻璃到代码复杂性

**P：** 最后，我想提一个更具思辨性的联想。代码空间有一个自旋玻璃特有的特征——**能量景观的粗糙性（ruggedness）**。在自旋玻璃中，微小的耦合常数变化可以彻底改变基态结构（混沌性）。在代码中，一个字符的变化可以把一个高分策略变成语法错误或完全不同的行为。

自旋玻璃理论（Parisi 的副本对称性破缺理论，2021年诺贝尔奖）告诉我们，对于这种粗糙景观：
- 基态是高度退化的（有很多几乎等能的局部最优）
- 这些局部最优之间被高能量的"势垒"分隔
- 系统在低温下呈现"超计量（ultrametric）"结构——局部最优之间的距离满足一种树形层次关系

如果代码空间也有类似的结构，那么：
- 存在很多"不同但同样好"的控制策略（高退化基态）
- 从一个好策略到另一个好策略需要经过"差策略"的过渡区域（势垒）
- 好的策略可能自然地组织成一棵"进化树"——一种层次结构

MAP-Elites 的多样性维护和岛屿模型的隔离进化，正好是处理这种粗糙景观的好策略——它们维护了多个不同"能量盆地"中的解，防止系统被困在单一盆地中。

**M：** 2024年的论文 "Exploring Loss Landscapes through the Lens of Spin Glass Theory" 和 2025年的 "A Spin Glass Characterization of Neural Networks" 正在沿这个方向走。他们用球形多自旋玻璃模型来分析神经网络的损失景观。我们完全可以把类似的分析应用到 policy-evolve 的适应度景观上——比如估算局部最优之间的"势垒高度"，或者检验解空间是否具有超计量结构。

### 3.7 联想：涨落-耗散定理与进化速率

**P：** 统计物理中有一个优美的结果——**涨落-耗散定理（Fluctuation-Dissipation Theorem, FDT）**。它说的是：系统对外部扰动的响应（耗散），与系统自身的热涨落成正比。在温度 T 下，响应函数 χ 和关联函数 C 的关系是：χ = C / kT。

在进化中，这有一个惊人的对应（Fisher 1930, "自然选择的基本定理"）：
$$\frac{d\bar{w}}{dt} = \text{Var}(w)$$

种群平均适应度的增长速率，等于种群中适应度的方差。

这意味着：**种群越多样（方差越大），进化得越快。** 这是多样性维护策略（MAP-Elites、岛屿模型、Novelty ELO）的理论基础——它们通过维护方差来维持进化速率。

而且注意：如果方差 → 0（mode collapse），进化速率 → 0。这是 mode collapse 有害性的精确量化——它不只是"看起来不好"，它从数学上杜绝了进一步改进的可能性。

## 第四章：更多可能的联想方向

### 4.1 重整化群与代码的层次结构

代码有天然的层次结构：字符 → 表达式 → 语句 → 函数 → 模块。这让人联想到重整化群（RG）——通过"粗粒化"来理解系统在不同尺度上的行为。

一个有趣的问题：能否对代码空间定义一种"重整化"操作——比如把低层细节（变量名、具体数值）积分掉，只保留高层结构（算法骨架）？这种操作可以帮助我们理解"哪些代码变异是本质的（改变了算法结构），哪些是无关紧要的（只是修改了常数）"。

### 4.2 对称性破缺与策略选择

HalfCheetah 的物理系统具有前后对称性（在某种近似意义上）。一个"好的步态"通常会打破这种对称性——前后腿在不同的相位运动。这是经典的**自发对称性破缺**。

在进化中，初始程序可能处于"对称相"（所有关节被同等对待），而一个好的进化结果通常处于"破缺相"（前后腿分化出不同的角色）。相变点就是这种分化开始发生的时刻。

### 4.3 临界性与"生命的边缘"

Stuart Kauffman 的"混沌边缘（Edge of Chaos）"理论认为，复杂适应系统在有序与混沌的相变点附近表现最优。在进化计算中，这对应于：

- **太有序**（低温）：种群收敛到局部最优，丧失适应性
- **太混乱**（高温）：变异太强，好的结构无法积累
- **边缘**（临界温度）：系统具有最大的"计算能力"——能同时维持结构和灵活性

policy-evolve 的最优运行状态可能恰好在这个"边缘"上。而退火调度的最佳策略可能不是单调降温，而是**维持在临界点附近**——通过监控比热容来自适应地调节温度。

### 4.4 信息论与麦克斯韦妖

LLM 在进化中扮演的角色，有点像麦克斯韦妖（Maxwell's Demon）——它"看"了大量代码（训练数据），获得了关于"什么是好代码"的信息，然后用这些信息来"引导"变异方向，降低搜索的有效"熵"。

但热力学第二定律告诉我们，妖精获取和处理信息本身需要消耗自由能（Landauer 原理）。在 policy-evolve 中，这对应于 LLM 推理的计算成本——每次调用 DeepSeek API 都需要消耗能量（电力、GPU 时间）。

所以 policy-evolve 的总"热力学效率"可以定义为：
$$\eta = \frac{\Delta F_{\text{evolve}}}{\text{total LLM compute cost}}$$

即：每单位 LLM 计算消耗，能让进化系统的自由能降低多少？这是一个可以量化比较不同进化策略效率的指标。

### 4.5 量子隧穿与架构变异

policy-evolve 中 5% 的架构变异率（全重写模式）在物理上类似于**量子隧穿**——粒子不是翻越势垒，而是"穿过"势垒。在代码空间中，diff-based 变异是"翻越"（沿能量面连续移动），而全重写是"隧穿"（跳到完全不同的代码区域）。

量子退火（Quantum Annealing）正是利用了这种机制——在 D-Wave 量子计算机中，量子隧穿效应允许系统逃脱经典模拟退火无法逃脱的深局部最优。policy-evolve 的架构变异率就是"隧穿概率"的工程对应物。

## 结语：两个世界的深层同构

这篇文档的核心论点是：**用 LLM 进化控制逻辑的系统，与平衡态统计物理之间的联系不是表面的类比，而是深层的数学同构。**

这种同构来自一个简单的事实：两个领域都在处理同一个数学问题——**在指数级大的状态空间中，如何高效地找到"好的"状态？** 统计物理用 Boltzmann 分布和蒙特卡洛方法；进化用适应度选择和变异算子；LLM 用 softmax 和 temperature 采样。它们是同一个问题的三张面孔。

配分函数 Z 编码了搜索空间的全局结构。自由能 F = -kT ln Z 给出了效率和多样性的最优平衡。相变标记了搜索动态的质变点。涨落-耗散定理将多样性和进化速率联系在一起。

policy-evolve 的工程师们已经直觉地使用了很多这些概念——Boltzmann 选择、多温度系统、岛屿隔离。这篇文档希望把这些直觉提升为系统的理论框架，并指出哪些地方还可以更深入地挖掘。

最终，我们相信：**统计物理不只是进化计算的一个比喻，它是进化计算的正确理论语言。**
