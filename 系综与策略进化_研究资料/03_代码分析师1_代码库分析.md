# policy-evolve 代码库深度分析报告（分析师#1）

## 1. 项目整体架构

### 1.1 项目目标
policy-evolve 是一个基于LLM驱动的进化策略优化系统，目标是为 MuJoCo HalfCheetah-v5 仿真环境进化出高性能的运动控制策略。系统使用 DeepSeek V3 作为代码进化引擎，Gemini 进行视频分析和优雅度评分，通过三岛 MAP-Elites 架构维持种群多样性。

### 1.2 两层架构
项目采用冻结上游 + 子类注入的双层设计：
- **上游层** (`src/openevolve/`): OpenEvolve 框架的逐字复制，视为只读
  - `controller.py` -- OpenEvolve 类
  - `database.py` -- ProgramDatabase (MAP-Elites + 岛模型)
  - `process_parallel.py` -- ProcessParallelController
  - `prompt/sampler.py` -- Prompt 构建器
  - `config.py` -- 配置数据类
- **实验层** (`src/experiment/variants/`): V4 扩展通过子类注入
  - `island_novelty_controller.py` -- IslandNoveltyOpenEvolve(OpenEvolve)
  - `island_novelty_database.py` -- IslandNoveltyProgramDatabase(ProgramDatabase)
  - `island_novelty_parallel.py` -- IslandNoveltyParallelController
  - `island_novelty_worker.py` -- 替代 worker 函数
  - `novelty_elo.py` -- NoveltyELOTracker

### 1.3 数据流
```
run.py -> runner.py -> IslandNoveltyOpenEvolve.run()
  +--> IslandNoveltyParallelController.start()
  +--> run_evolution() 主循环:
       +-- 对每个 island 轮流:
       |     1. database.sample_from_island(island_id) -> (parent, inspirations)
       |     2. _create_database_snapshot() -> 序列化数据库状态
       |     3. executor.submit(island_novelty_run_iteration_worker, ...)
       +-- Worker 进程内部:
       |     1. 构建 prompt
       |     2. LLM 生成代码变异 (diff-based)
       |     3. 应用 diff -> child_code
       |     4. evaluator.evaluate_program(child_code)
       |     5. 返回 SerializableResult
       +-- 主进程收到结果后:
             1. NoveltyELO insert_incremental(child)
             2. database.add(child, target_island=...)
             3. 定期: full_rerank() / migrate_programs()
             4. checkpoint & early_stopping
```

## 2. LLM调用机制

- LLM: DeepSeek V3, temperature=0.8, top_p=0.95, max_tokens=4096
- Prompt: system message(角色定义) + user message(代码+metrics+history+inspirations)
- 变异: diff-based (SEARCH/REPLACE块) + 5%架构变异(全重写)
- 交叉: 通过inspirations隐式实现 + cross_island_inspiration_count=1

## 3. 选择策略

三模态策略:
- Exploration (0.2): 随机选取
- Exploitation (0.7): 从精英库选取
- Random (0.1): 全局随机

V4新增 temperature-softmax 采样(sampling_temperature=1000): rank-based softmax

## 4. 适应度评估

级联评估: Stage 1(3 episodes快速) → Stage 2(5 episodes + video + Gemini)
核心适应度: combined_score = mean_reward (velocity - 0.1*ctrl_cost)

## 5. 种群管理

- MAP-Elites: complexity维度, cell_capacity=15, 7个bin
- 2岛: Island 0(fitness) + Island 1(novelty_elo)
- 迁移: 每50代, 迁移率10%
- Novelty ELO: 三轮漏斗 + Swiss锦标赛 + LLM judge

## 6. 与统计物理的映射

### 已有的统计物理结构:
1. `calculate_softmax_heat()` (island_novelty_sampling.py:8-21) = Boltzmann分布
2. `expected_score()` (novelty_elo.py:30-32) = Fermi-Dirac分布 (β=ln(10)/400≈0.00576)
3. 双系综线性混合 (island_novelty_database.py:442-505) = 广义Gibbs系综特殊情况
4. 多岛系统 = parallel tempering
5. 配分函数Z在calculate_softmax_heat中被计算但被丢弃

### 4个温度参数的物理角色:
- LLM temperature (0.8) = 热噪声强度
- Sampling temperature (1000) = 选择温度(极高→均匀分布)
- Novelty judge temperature (0.1) = 测量温度(近确定性)
- Reward temperature (动态) = 混合系综温度

### 5个隐式能量函数:
- E = -combined_score (fitness选择)
- E = -elo_score (novelty ELO)
- E = -elegance_total (优雅度)
- Cell competition内的fitness
- Population culling的global fitness
